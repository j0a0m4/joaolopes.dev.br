Mobile audit: what I found

5 concrete issues across phone-size viewports (360–430px). 768px+ already looks great.

Issue 1 — Critical

Nav logo wraps to 2 lines

At ≤430px, "João Lopes" breaks onto a second line because the flex nav tries to fit logo + 4 links in one row. The nav grows to ~2× its intended height, pushing content down.

Issue 3 — High

Code blocks scroll horizontally with no visual cue

overflow-x:auto is set (good), but there's no fade or indicator that content continues to the right. Long lines silently disappear, and touch scroll on code blocks can conflict with page scroll.

Issue 4 — Medium

Dateline wraps oddly on narrow screens

"Published 2026-03-17 · Updated 2026-03-18" breaks mid-sentence to a second line at 373px. Minor but looks unpolished.

Issue 5 — Medium

TOC takes up half the first screen on mobile

On this post the TOC is 10 items tall — ~400px on mobile. Readers who want to start reading immediately scroll past it. No collapse option exists.

What's already working

768px+ (iPad, Z Fold 6 inner) looks excellent — no changes needed there. Body text is well-sized at 18px. Reading width (42rem) is comfortable. Light/dark mode is solid.

First question: the nav on mobile

The broken nav is the most visible issue. There are two ways to fix it:

A

Collapse to hamburger menu

Logo stays left, ☰ button right. Tap to reveal Posts / Tags / About / RSS in a dropdown. Common pattern, clean on small screens. Adds a small JS requirement.

B

Stack: logo on top, links below

Nav becomes 2 rows at small widths — logo row + links row. Pure CSS, no JS. Simpler but uses more vertical space than today.

C

Shrink the logo text

Keep single-row layout but reduce the logo font size and let the nav compress. Trade-off: logo becomes small/harder to tap.
