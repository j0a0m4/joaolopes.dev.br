Design: Sections 2 & 3 — TOC + CSS fixes

Section 2: collapsible TOC. Section 3: the five CSS-only bug fixes.

Section 2 — Collapsible TOC

Native <details>/<summary> — zero JS, zero ARIA

The existing .toc <nav> wraps in a <details>. The <summary> becomes the "Contents" label. Screen readers announce it as expandable natively — no ARIA authoring needed.

layout.clj change — toc-nav becomes
;; details = native disclosure, no JS needed
[:details.toc-details {:open true} ; open on desktop via CSS
 [:summary.toc-label "Contents"]
 [:ul ...]]
CSS — desktop stays open, mobile collapses
/* Desktop: always open, hide the triangle */
.toc-details[open] { display: block; }
.toc-details > summary { list-style: none; cursor: default; }

/* Mobile: collapsed by default, show toggle affordance */
@media (max-width: 480px) {
  .toc-details:not([open]) > summary::after {
    content: " ▸ " attr(data-count);
    color: var(--muted);
  }
  .toc-details > summary { cursor: pointer; }
}

A11y note: <details>/<summary> is ARIA role group/button implicitly. VoiceOver announces "Contents, collapsed, button" — no manual ARIA needed. The open attribute on the server-rendered HTML means desktop gets it open without JS.

Section 3 — Five CSS fixes (all in one @media (max-width: 480px) block)

① Tags wrap
flex-wrap: wrap on .tags — one line. Last tag is no longer clipped.
② Code block overflow indicator
A ::after pseudo-element on pre adds a right-edge fade gradient when the block overflows. Signals "there's more here" without obscuring content.
③ Dateline wraps gracefully
.post-dateline gets display: flex; flex-wrap: wrap; gap: 0.25rem — "Published …" and "· Updated …" stack cleanly when narrow instead of mid-sentence wrap.
④ Body padding
body gets padding: 1.25rem 1rem on mobile (down from 2rem 1rem) — reduces wasted vertical space at top.
⑤ Nav links touch targets
.nav-links a gets padding: 0.5rem 0 when the menu is open — each link has a ≥44px tap target height (WCAG 2.5.5).

What stays unchanged

  • Font size (18px) — already good on mobile
  • Max-width (42rem) — only kicks in on wider screens
  • All 768px+ layouts — no changes, already solid
  • Colors, theme, SVG diagrams — untouched
