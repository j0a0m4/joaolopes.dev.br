Design: Section 1 of 3 — Inline diagram wrapper

How SVGs in posts change: from a raw <svg> dump to a semantic, accessible, clickable figure.

Current HTML (after inlining)

What the post renders today
<!-- raw SVG, no wrapper, no a11y, no click -->
<svg xmlns="..." width="100%" viewBox="0 0 760 220">
  ...
</svg>

New HTML (produced by updated inline-svgs in markdown.clj)

What the post renders after the change
<!-- semantic figure, a11y SVG title, clickable caption -->
<figure class="diagram-figure">
  <a href="/diagrams/agent-loop/" class="diagram-link" aria-label="View Agent Loop diagram">
    <svg role="img" aria-labelledby="diag-agent-loop-title" width="100%" viewBox="0 0 760 220">
      <title id="diag-agent-loop-title">Agent Loop</title>
      ...
    </svg>
  </a>
  <figcaption>
    <a href="/diagrams/agent-loop/" class="diagram-caption-link">View Agent Loop diagram →</a>
  </figcaption>
</figure>

A11y contract

  • role="img" on SVG — tells screen readers "this is an image", not a group of shapes
  • <title> + aria-labelledby — screen reader announces "Agent Loop, image" when focused
  • <a> wraps SVG — keyboard focusable, one Tab stop, navigates to diagram page
  • aria-label on the link — overrides link text for SR: "View Agent Loop diagram" (not just "link")
  • <figcaption> — visible text for all users, screen readers associate it with the figure
  • cursor: pointer + hover overlay via CSS — purely decorative, no semantic role
